## **Design Patterns Cheat Sheet (from our discussion)**

| Pattern                     | Problem                                                                        | Solution                                                                               | Real-World Analogy                                        | Example (Description)                                      |
| --------------------------- | ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- | --------------------------------------------------------- | ---------------------------------------------------------- |
| **Singleton**               | Ensure only one instance of a class exists                                     | Provide a global access point and prevent multiple instantiations                      | Prime Minister: only one in the country                   | Python metaclass ensuring single instance                  |
| **Factory Method**          | Need flexibility to create objects without specifying exact class              | Define an interface for creating objects; subclasses decide which class to instantiate | Logistics company creating different types of documents   | Document factory producing PDFs, Word docs                 |
| **Abstract Factory**        | Need to create families of related objects without specifying concrete classes | Provide an interface to create related objects                                         | Furniture factory producing modern or Victorian furniture | GUI factory creating buttons and menus                     |
| **Builder**                 | Complex object creation step-by-step                                           | Separate construction from representation                                              | Burger assembly line: add ingredients step-by-step        | Burger builder with chaining methods                       |
| **Prototype**               | Need to create new objects by copying existing ones                            | Clone an existing object rather than creating from scratch                             | Photocopy machine: duplicate documents                    | Cloning objects with copy method                           |
| **Adapter**                 | Need to make incompatible interfaces work together                             | Wrap an object with a compatible interface                                             | Travel plug adapter converting socket types               | XML-to-JSON converter                                      |
| **Bridge**                  | Decouple abstraction from implementation so both can vary independently        | Separate abstraction and implementation into two hierarchies                           | TV remote controlling different devices                   | Shape-Color example with remote controlling TV/Device      |
| **Composite**               | Represent part-whole hierarchies                                               | Compose objects into tree structures                                                   | File system: folders and files                            | Folder-File tree example                                   |
| **Decorator**               | Need to add responsibilities to objects dynamically                            | Wrap objects with decorators to add behavior                                           | Coffee with multiple toppings                             | Dynamic behavior: coffee with milk, sugar, etc.            |
| **Facade**                  | Simplify complex subsystems                                                    | Provide a unified interface                                                            | Hotel concierge / Home Theater control                    | Simplified interface to complex subsystem                  |
| **Flyweight**               | Avoid memory overhead from many similar objects                                | Share common parts of objects                                                          | Airplane seat sharing / text editor characters            | Memory-efficient objects like shared characters            |
| **Proxy**                   | Control access or add behavior without changing the real object                | Provide a placeholder that controls access                                             | Credit card / lazy image loading                          | Access-control proxy for an image loader                   |
| **Strategy**                | Need to switch algorithms dynamically                                          | Encapsulate algorithms into interchangeable classes                                    | Payment methods or navigation routes                      | Navigator selecting CarRoute, WalkRoute, BikeRoute         |
| **Observer**                | One-to-many dependency: update objects when another changes                    | Subject notifies multiple observers automatically                                      | YouTube subscriptions                                     | Stock price change notifying investors                     |
| **Command**                 | Need to encapsulate requests as objects                                        | Commands encapsulate requests; invoker executes them                                   | Remote control buttons                                    | Remote controlling light on/off with undo                  |
| **Iterator**                | Need sequential access without exposing collection internals                   | Provide an iterator interface                                                          | Book reader flipping pages                                | Custom BookCollection iterator                             |
| **Mediator**                | Many objects interacting tightly                                               | Centralize communication through a mediator                                            | Air traffic control tower                                 | ChatRoom coordinating messages between users               |
| **Memento**                 | Need to save/restore state without breaking encapsulation                      | Capture object state in mementos                                                       | Game save/load checkpoints                                | TextEditor saving/restoring content for undo               |
| **State**                   | Object behavior depends on its state                                           | Encapsulate states into classes; context delegates behavior                            | Music player: play/pause button                           | MusicPlayer switching between PlayingState and PausedState |
| **Template Method**         | Algorithm structure is fixed but some steps vary                               | Define skeleton in base class; subclasses implement steps                              | Recipe with fixed steps and customizable ingredients      | DataParser skeleton: read → process → save                 |
| **Visitor**                 | Need to perform operations on object structure without changing classes        | Separate operation logic into visitors; elements accept visitor                        | Tax visitor evaluating different employee types           | SpellChecker and Renderer visiting Paragraph and Image     |
| **Interpreter**             | Need to evaluate or interpret expressions in a language                        | Define grammar classes; each interprets itself                                         | Calculator evaluating expressions                         | Arithmetic expression: (5+3)-2 using Number, Add, Subtract |
| **Chain of Responsibility** | Sender of request shouldn’t know which handler will process it                 | Pass request along a chain until handled                                               | Customer support escalation                               | Level 1 → Level 2 → Level 3 support handling tickets       |
